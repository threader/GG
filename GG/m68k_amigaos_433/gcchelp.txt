+++++++++++++++++++++++++++++++++++ now come diffrence that happen on gcc >2.0 ++++++++++++++++++++

The paste operator work diffrent in GCC > 2.0. in gcc2.95 this work ok

#define CONSTRUCTOR(name,pri) \
VOID __ctor_##name##(VOID)

CONSTRUCTOR(mylib_lib)
{

}

but on compilers > 2.0 you get this error

error: pasting "__ctor_mylib_lib" and "(" does not give a valid preprocessing token

to avoid this remove the paste ## command.

#define CONSTRUCTOR(name,pri) \
VOID __ctor_##name(VOID)

thats correct for compilers >2.

Problem with 
error: asm specifer for variable __RawPutChar_c conflicts with asm clobber list.

#define RawPutChar(c) ({ \
  ULONG _RawPutChar_c = (c); \
  { \
  register struct Library * const __RawPutChar__bn __asm("a6") = SysBase;\
  register ULONG __RawPutChar_c __asm("d0") = (_RawPutChar_c); \
  __asm volatile ("jsr a6@(-516:W)" \
  : \
  : "r"(__RawPutChar__bn), "r"(__RawPutChar_c) \
  : "d0" "d1", "a0", "a1", "fp0", "fp1", "cc", "memory"); \
  } \
})

this happen because d0 is in clobber list but is used as register for variable __RawPutChar_c.

correct is  

#define RawPutChar(c) ({ \
  ULONG _RawPutChar_c = (c); \
  { \
  register struct Library * const __RawPutChar__bn __asm("a6") = SysBase;\
  register ULONG __RawPutChar_c __asm("d0") = (_RawPutChar_c); \
  __asm volatile ("jsr a6@(-516:W)" \
  : \
  : "r"(__RawPutChar__bn), "r"(__RawPutChar_c) \
  : "d1", "a0", "a1", "fp0", "fp1", "cc", "memory"); \
  } \
})

if the gcc assembler bring an strange error or crash it happen because  a \n\ is miss at the end of the asm line.(also need \n if only 1 asm line )

usefull gcc flags:

-v show version of gcc and give more precise info about include path etc
-S only asm output in current .o dest file
-E output of preprocessor in ascii (expandet macros) in current .o dest file
-O0 set optimizer to 0 -O3 optimizer set to 3.
-g debug build
-s avoid debug symbol build
-V choose the compiler Version for build (-V 3.4.0 use 3.4.0 compiler)
-b choose the target machine (-b m68k-amigaos) use

add own includes to usr/local/amiga/m68k-amigaos/sys-include

if you get __decimalpoint miss linker error use -noixemul linker flag or when you want use ixemul link with libc.a
